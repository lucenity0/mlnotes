UNIT I

Introduction To Machine Learning

ML Landscape · Types · Batch/Online · Instance/Model-Based · Challenges · End-to-End Projects

What is Machine Learning?

Machine Learning is a core sub-area of Artificial Intelligence (AI) where systems learn from data and improve their performance on tasks without being explicitly programmed. Instead of writing fixed rules, we feed data and let the machine discover patterns.

Formally, Arthur Samuel (1959) defined ML as: "the field of study that gives computers the ability to learn without being explicitly programmed."

Tom Mitchell (1997) gave a formal definition: "A computer program is said to learn from experience E with respect to some task T and some performance measure P, if its performance on T, as measured by P, improves with experience E."

KEY INSIGHT

ML is at the intersection of Statistics, Computer Science, and Artificial Intelligence. It is also known as predictive analytics or statistical learning.

APPLICATIONS
  • Spam email detection
  • Medical image diagnosis (tumor detection)
  • Recommendation systems (Netflix, Amazon)
  • Self-driving cars, chatbots, fraud detection

  • Fluctuating environments where data patterns shift over time
  • Getting insights about complex datasets (data mining)
Types of Machine Learning
1. Supervised Learning

The training data includes desired outputs (labels). The algorithm learns to map inputs to outputs.

  • Classification: Predicting a category (spam/not spam, disease/healthy)
  • Regression: Predicting a continuous value (house price, stock price)

Algorithms: Linear Regression, Logistic Regression, Decision Trees, SVM, KNN, Neural Networks, Random Forest

Examples: Spam detection, tumor classification, stock price prediction, sentiment analysis

2. Unsupervised Learning

The model is trained on unlabeled data. The goal is to discover hidden patterns and structures.

  • Clustering: K-Means, DBSCAN, Hierarchical Clustering
  • Dimensionality Reduction: PCA, t-SNE, Autoencoders
  • Association Rules: Apriori, Eclat (market basket analysis)
  • Anomaly Detection: One-class SVM, Isolation Forest

Examples: Customer segmentation, anomaly detection in cybersecurity, recommender systems

3. Semi-supervised Learning

Uses a small portion of labeled data combined with a large amount of unlabeled data. The labeled data trains an initial model, which then assigns pseudo-labels to unlabeled data, and both are used together.

Example: A hospital has 10,000 patient X-rays but only 500 are labeled by experts (expensive). Semi-supervised learning uses both labeled and unlabeled records.

4. Reinforcement Learning

An agent learns to make decisions by interacting with an environment, receiving rewards for good actions and penalties for bad ones. The goal is to maximize cumulative reward.

Components: Agent, Environment, State (S), Action (A), Reward (R), Policy (π), Value Function (V)

Examples: Training game-playing AIs (AlphaGo), robotic control, self-driving cars

Batch Learning vs. Online Learning
BATCH LEARNING
  • Model trained on the entire dataset at once
  • Once trained, it does not update until retrained
  • Also called offline learning
  • Requires significant computing resources during training
  • Model launched into production unchanged
  • Use when: Data is static and complete
ONLINE LEARNING
  • Model updates incrementally as new data arrives
  • Great for streaming data (stock prices, weather)
  • Can discard old data after learning
  • Can train on huge datasets that don't fit in memory
  • Learning rate controls how fast it adapts
  • Use when: Data is continuous and changing
Learning Rate in Online Learning: High learning rate → adapts quickly but forgets old data. Low learning rate → more stable but slower to adapt. Setting the right learning rate is crucial.
Instance-Based vs. Model-Based Learning
INSTANCE-BASED

The model memorizes training examples and predicts by comparing new inputs to stored instances using a similarity measure (lazy learning).

Examples: K-Nearest Neighbor (KNN), Case-Based Reasoning

Advantage: Simple, no explicit training needed
Disadvantage: Slow at prediction time, needs all data in memory

MODEL-BASED

Learns an internal representation (mathematical model) from training data and uses it for predictions. Generalizes from data by fitting parameters.

Examples: Linear Regression, Neural Networks, Decision Trees

Advantage: Fast at prediction, compact storage
Disadvantage: Requires careful model selection

Main Challenges of Machine Learning
1. Insufficient Quantity of Training Data

ML algorithms need lots of data to learn effectively — thousands of examples for simple problems, millions for complex tasks like image/speech recognition. With too little data, models cannot generalize well.

Example: Training a deep learning model for rare bird species with only 100 images per species will lead to poor accuracy on new data.

2. Non-Representative Training Data

If training data doesn't represent the real-world cases, the model is biased. This leads to poor generalization — either from sampling noise (too little data) or sampling bias (systematic flaws).

Example: A facial recognition system trained only on light-skinned faces fails on darker-skinned individuals.

3. Poor-Quality Data

Data full of errors, outliers, and noise makes it harder to find underlying patterns. Solutions include discarding or fixing outliers, filling missing values, and cleaning noise.

  • Errors: Wrong values (age = 200, temperature = -500°C)
  • Outliers: Unusual but possible (income = ₹50 crore)
  • Noise: Random variation in sensor readings
4. Irrelevant Features (Feature Engineering)

"Garbage in, garbage out." Success depends on having good features — not too many irrelevant ones. Feature engineering involves: feature selection, feature extraction, and creating new features.

Example: Building a spam classifier but including font color instead of word features leads to poor results.

5. Overfitting

The model learns training data too well — including noise and random fluctuations — and fails to generalize to new data. Model is too complex relative to data.

  • Low training error, high test error
  • Solutions: Regularization, simpler model, more training data, dropout
6. Underfitting

The model is too simple to capture the complexity in the data. Poor performance on both training and test data.

  • High training error AND high test error
  • Solutions: More complex model, better features, reduce regularization
Bias-Variance Tradeoff: Underfitting = High Bias. Overfitting = High Variance. The goal is to find the sweet spot — a model complex enough to capture patterns, but not so complex that it memorizes noise.
End-to-End Machine Learning Project
  1. Frame the Problem: Define goals (classification/regression?), success metrics, existing solutions
  2. Get the Data: Download from sources like Kaggle, UCI, HuggingFace; check data size and type
  3. Explore & Visualize the Data: Histograms, scatter plots, correlation matrices, look for patterns and anomalies
  4. Prepare the Data: Handle missing values, encoding, normalization/standardization, feature engineering
    • MinMax Scaler: Scales to [0,1] range. X_scaled = (X - X_min)/(X_max - X_min)
    • Standard Scaler: Scales to mean=0, std=1. X_scaled = (X - μ)/σ
  5. Select & Train Models: Try multiple models (linear, tree-based, ensemble) with cross-validation
  6. Fine-Tune Models: Grid search / random search for hyperparameters; use validation set
  7. Evaluate on Test Set: Final performance check on held-out test data
  8. Present & Deploy Solution: Document, deploy, monitor for data drift

Linear Regression

Linear regression models the linear relationship between a dependent variable Y and one or more independent variables X. It is used for regression tasks (predicting continuous values).

Simple Linear Regression
Equation
Y = β₀ + β₁X + ε Where: Y = Dependent variable (target) X = Independent variable (feature) β₀ = Intercept (value of Y when X=0) β₁ = Slope (change in Y per unit change in X) ε = Error term (residual)
Finding β₀ and β₁ (Ordinary Least Squares)
β₁ = Σ(Xᵢ - X̄)(Yᵢ - Ȳ) / Σ(Xᵢ - X̄)² β₀ = Ȳ - β₁ · X̄ Also: β₁ = [n·ΣXᵢYᵢ - ΣXᵢ·ΣYᵢ] / [n·ΣXᵢ² - (ΣXᵢ)²]
Solved Example — Predict 7th and 9th week sales

Given: Week (X): 1,2,3,4,5 | Sales in thousands (Y): 1,3,4,8,10

Step 1: Compute means
X̄ = (1+2+3+4+5)/5 = 3, Ȳ = (1+3+4+8+10)/5 = 5.2
Step 2: Compute β₁
Σ(Xᵢ-X̄)(Yᵢ-Ȳ) = (1-3)(1-5.2)+(2-3)(3-5.2)+(3-3)(4-5.2)+(4-3)(8-5.2)+(5-3)(10-5.2)
= (-2)(-4.2)+(-1)(-2.2)+(0)(-1.2)+(1)(2.8)+(2)(4.8) = 8.4+2.2+0+2.8+9.6 = 23
Σ(Xᵢ-X̄)² = 4+1+0+1+4 = 10
β₁ = 23/10 = 2.3
Step 3: Compute β₀
β₀ = Ȳ - β₁·X̄ = 5.2 - 2.3×3 = 5.2 - 6.9 = -1.7
Step 4: Regression Equation
Y = -1.7 + 2.3X
Prediction:
Week 7: Y = -1.7 + 2.3(7) = -1.7 + 16.1 = 14.4 thousand
Week 9: Y = -1.7 + 2.3(9) = -1.7 + 20.7 = 19 thousand
Error Metrics
MSE (Mean Squared Error) = (1/n) Σ(Yᵢ - Ŷᵢ)² RMSE (Root MSE) = √MSE MAE (Mean Absolute Error) = (1/n) Σ|Yᵢ - Ŷᵢ| MAPE (Mean Abs. % Error) = (100/n) Σ|Yᵢ - Ŷᵢ|/Yᵢ
Multiple Linear Regression

Extension of simple linear regression with multiple independent variables.

Y = β₀ + β₁X₁ + β₂X₂ + ... + βₖXₖ + ε Matrix Form: β = (XᵀX)⁻¹ Xᵀy Where X is the design matrix (with column of 1s for intercept) y is the target vector β is the coefficient vector
Solved Example — Multiple Linear Regression (Matrix Method)

Given: x1=[60,62,67,70,71,72,75,78], x2=[22,25,24,20,15,14,14,11], y=[140,155,159,179,192,200,212,215]

Step 1: Construct design matrix X
Add a column of 1s for intercept: X = [1, x1, x2] (8×3 matrix)
Step 2: Compute XᵀX and Xᵀy
These are matrix multiplications. X is 8×3, so XᵀX is 3×3 and Xᵀy is 3×1.
Step 3: Apply Normal Equation
β = (XᵀX)⁻¹ · Xᵀy → yields β₀, β₁, β₂
The result gives the fitted equation: Y = β₀ + β₁·x1 + β₂·x2
Key Formula: β = (XᵀX)⁻¹Xᵀy — this minimizes sum of squared residuals
Logistic Regression

Logistic regression is used for classification (not regression despite the name!). It predicts a probability that an instance belongs to a class, then converts to binary outcome.

Sigmoid Function (Logistic Function)
σ(z) = 1 / (1 + e⁻ᶻ) Where z = β₀ + β₁X₁ + β₂X₂ + ... Output P(y=1|x) = σ(z) — always between 0 and 1 Decision Rule: If P ≥ 0.5 → Class 1; If P < 0.5 → Class 0
Solved Example — Student Pass/Fail Prediction

Model: β₀ = -6, β₁ = 0.9 (study hours). Student studies 7 hours. Threshold = 0.6

Step 1: Compute z = β₀ + β₁·X = -6 + 0.9×7 = -6 + 6.3 = 0.3
Step 2: Apply sigmoid: P = 1/(1 + e⁻⁰·³) = 1/(1 + 0.7408) = 1/1.7408 ≈ 0.5744 (57.44%)
Step 3: Compare to threshold 0.6: 0.5744 < 0.6 → Predicted: FAIL

K-Nearest Neighbors (KNN)

KNN is a lazy, instance-based classifier that classifies new data points based on the majority class among its K nearest neighbors. No training phase — just memorizes data.

Distance Metric
Euclidean Distance = √(Σ(Xᵢ - Yᵢ)²) = √((X₁-X₂)² + (Y₁-Y₂)²) [2D case] Manhattan Distance = Σ|Xᵢ - Yᵢ|
KNN Algorithm Steps
  1. Assign a value to K
  2. Calculate the Euclidean distance from the new point to all training points
  3. Sort distances in ascending order
  4. Pick the K nearest neighbors
  5. For classification: assign the majority class among K neighbors
    For regression: average the K neighbors' values
Solved Example — Diabetic Patient Prediction (K=3)

Predict Sugar for BMI=43.6, Age=40 using K=3.

Step 1: Compute distances from (43.6, 40) to all points:
d(33.6, 50) = √((43.6-33.6)² + (40-50)²) = √(100+100) = √200 ≈ 14.14
d(43.1, 67) = √((43.6-43.1)² + (40-67)²) = √(0.25+729) = √729.25 ≈ 27.00
d(35.3, 23) = √((43.6-35.3)² + (40-23)²) = √(68.89+289) = √357.89 ≈ 18.92
d(35.9, 67) = √(59.29+729) ≈ 28.08 | d(36.7, 45) = √(47.61+25) ≈ 10.70
Step 2: Sort and pick K=3 nearest:
1st: (36.7, 45) → Sugar=1, dist=10.70
2nd: (33.6, 50) → Sugar=1, dist=14.14
3rd: (35.3, 23) → Sugar=1, dist=18.92
Step 3: Majority vote: All 3 neighbors have Sugar=1 → Predicted: Sugar = 1 (diabetic)
Choosing K: Small K → sensitive to noise. Large K → includes points from other classes. Rule of thumb: K = √n (where n = number of training samples). Always use odd K for binary classification to avoid ties.
KNN Algorithm Pseudocode
Algorithm KNN_Classify(X_train, y_train, x_new, K): distances = [] FOR each (xᵢ, yᵢ) in training data: d = euclidean_distance(x_new, xᵢ) distances.append((d, yᵢ)) Sort distances ascending by d k_neighbors = first K elements of distances class_votes = count(labels in k_neighbors) RETURN argmax(class_votes)
Decision Trees (ID3 Algorithm)

A decision tree is a tree-like model where each internal node represents a test on a feature, each branch represents an outcome, and each leaf represents a class label (classification) or value (regression).

Entropy

Entropy measures the impurity/uncertainty in a dataset. If all examples belong to one class, entropy = 0 (pure). If classes are equally split, entropy = 1 (maximum uncertainty).

H(S) = -Σ pᵢ · log₂(pᵢ) For binary classification with p positive, q negative examples: H(S) = -p·log₂(p) - q·log₂(q) Special cases: H = 0 when all examples same class (pure node) H = 1 when 50% positive, 50% negative
Information Gain

Information Gain (IG) measures the reduction in entropy after splitting on an attribute. The attribute with the highest IG is chosen as the split point.

IG(S, A) = H(S) - Σ (|Sᵥ|/|S|) · H(Sᵥ) Where: S = current dataset A = attribute being evaluated Sᵥ = subset of S where attribute A has value v H(S) = entropy of S
Solved Example — ID3 on Play Tennis Dataset (14 examples: 9+, 5-)
Step 1: Entropy of S
H(S) = -9/14·log₂(9/14) - 5/14·log₂(5/14) = -(0.643)(−0.637) − (0.357)(−1.485) = 0.410 + 0.530 = 0.940
Step 2: IG for Outlook (Sunny:5, Overcast:4, Rain:5)
H(Sunny) = -2/5·log₂(2/5) - 3/5·log₂(3/5) = 0.971
H(Overcast) = -4/4·log₂(4/4) = 0 [all positive → pure!]
H(Rain) = -3/5·log₂(3/5) - 2/5·log₂(2/5) = 0.971
IG(Outlook) = 0.940 - [5/14×0.971 + 4/14×0 + 5/14×0.971] = 0.940 - 0.694 = 0.246
Step 3: Compare all IG values
IG(Outlook) = 0.246 ← Highest → Root node!
IG(Humidity) = 0.151
IG(Wind) = 0.048
IG(Temperature) = 0.029
Step 4: Select Outlook as root. Overcast branch is pure (all Yes) → leaf node. Recursively split Sunny and Rain branches.
Solved Example — Decision Tree for GPA/Studied → Passed (from Dec 2025 paper)

Dataset: GPA (L/M/H), Studied (T/F), Passed (T/F) — 6 examples

Distribution: 4 Pass (T), 2 Fail (F)
H(Passed) = -4/6·log₂(4/6) - 2/6·log₂(2/6) = -(0.667)(−0.585) − (0.333)(−1.585) = 0.390 + 0.528 = 0.918
H(Passed | GPA):
GPA=L: {F,T} → 1T, 1F → H = 1.0 (2 examples)
GPA=M: {F,T} → 1T, 1F → H = 1.0 (2 examples)
GPA=H: {T,T} → 2T, 0F → H = 0.0 (2 examples, pure!)
H(Passed|GPA) = 2/6×1.0 + 2/6×1.0 + 2/6×0.0 = 0.333 + 0.333 = 0.667
IG(GPA) = 0.918 - 0.667 = 0.251
H(Passed | Studied):
Studied=F: {F,F,T} → 1T, 2F → H = -1/3·log₂(1/3) - 2/3·log₂(2/3) = 0.918 (3 examples)
Studied=T: {T,T,T} → 3T, 0F → H = 0.0 (pure!)
H(Passed|Studied) = 3/6×0.918 + 3/6×0.0 = 0.459
IG(Studied) = 0.918 - 0.459 = 0.459 ← Higher!
→ Root node = Studied
Regression Trees

When the target is continuous (not categorical), we build a Regression Tree. Instead of entropy/information gain, we use Standard Deviation Reduction (SDR):

SDR(A) = SD(S) - Σ (|Sᵥ|/|S|) · SD(Sᵥ) Split criterion: choose attribute A with maximum SDR Leaf value: mean of target values in that leaf
Support Vector Machines (SVM)

SVM is a supervised learning algorithm for classification and regression. It finds the optimal hyperplane that maximizes the margin between two classes.

Key Concepts
HYPERPLANE

A decision boundary separating classes. In 2D it's a line, in 3D a plane, in nD a hyperplane. Equation: wᵀx + b = 0

MARGIN

Distance between the hyperplane and the nearest data points from each class. SVM maximizes this margin → better generalization.

SUPPORT VECTORS

The data points closest to the hyperplane. These are the critical points — removing others doesn't change the decision boundary.

SVM Decision Function: f(x) = sign(wᵀx + b) Constraints for correct classification: wᵀxᵢ + b ≥ +1 for yᵢ = +1 (positive class) wᵀxᵢ + b ≤ -1 for yᵢ = -1 (negative class) Or equivalently: yᵢ(wᵀxᵢ + b) ≥ 1 Margin = 2/||w|| Optimization: Maximize margin = Minimize (1/2)||w||²
Linear SVM vs Non-Linear SVM
FeatureLinear SVMNon-Linear SVM
DataLinearly separableNot linearly separable
BoundaryStraight hyperplaneComplex boundary via kernel trick
KernelLinear (no transformation)RBF, Polynomial, Sigmoid
Kernel Functions
Linear Kernel: K(x,y) = xᵀy Polynomial Kernel: K(x,y) = (xᵀy + c)ᵈ RBF (Gaussian): K(x,y) = exp(-γ||x-y||²) Sigmoid Kernel: K(x,y) = tanh(αxᵀy + c) The "Kernel Trick": Maps data into higher-dimensional space without explicitly computing the transformation. Makes non-linearly separable data linearly separable in higher dimensions.
Hard Margin vs. Soft Margin SVM
HARD MARGIN

No misclassification allowed. Requires data to be perfectly linearly separable. Very strict — fails when data has noise or overlap.

SOFT MARGIN

Allows some misclassification. Uses slack variables (ξᵢ) to permit points inside or on the wrong side of the margin. Controlled by hyperparameter C — high C → less tolerance for errors.

Solved Example — SVM Weight Vector and Hyperplane (from Aug 2025 paper)

Positive SVs: {(3,1),(3,-1),(6,1),(6,-1)} | Negative SVs: {(1,0),(0,1),(0,-1),(-1,0)}

Support vectors identified: S₁=(1,0), S₂=(3,1), S₃=(3,-1)
Augmented with bias: S̃₁=(1,0,1), S̃₂=(3,1,1), S̃₃=(3,-1,1)
Set up system of equations (KKT conditions):
α₁(S̃₁·S̃₁) + α₂(S̃₂·S̃₁) + α₃(S̃₃·S̃₁) = -1 → 2α₁ + 4α₂ + 4α₃ = -1
α₁(S̃₁·S̃₂) + α₂(S̃₂·S̃₂) + α₃(S̃₃·S̃₂) = +1 → 4α₁ + 11α₂ + 9α₃ = 1
α₁(S̃₁·S̃₃) + α₂(S̃₂·S̃₃) + α₃(S̃₃·S̃₃) = +1 → 4α₁ + 9α₂ + 11α₃ = 1
Solving gives: α₁ = -3.5, α₂ = 0.75, α₃ = 0.75
Weight vector: w̃ = Σαᵢ·S̃ᵢ = -3.5(1,0,1) + 0.75(3,1,1) + 0.75(3,-1,1) = (1,0,-2)
So w = (1,0) and b = -2 → Hyperplane: x₁ - 2 = 0 → x₁ = 2 (vertical line at x=2)
UNIT III

Design & Analysis of ML Experiments

Guidelines · Cross Validation · Performance Metrics · Statistical Tests

Guidelines for Machine Learning Experiments

A systematic approach ensures reliable and reproducible results when assessing or comparing ML algorithms.

  1. Aim of the Study: Clearly define the problem. Goals: estimate error, compare two algorithms, rank multiple algorithms. Define success criteria upfront.
  2. Selection of Response Variable: For classification → misclassification error (0/1 loss); For regression → MSE; Also consider Precision, Recall, F1 for information retrieval tasks.
  3. Choice of Factors and Levels: Hyperparameter tuning → parameters are factors; Algorithm comparison → the algorithm itself is a factor. Carefully select levels to avoid missing optimal configurations.
  4. Choice of Experimental Design: Use factorial design to capture interactions between factors. Consider data size, train-test splits, and resampling strategies.
  5. Performing the Experiment: Conduct trial runs first. Save random seeds for reproducibility. Use well-tested libraries.
  6. Statistical Analysis of Data: Use hypothesis testing to validate claims. Visualize error distributions. Never make subjective interpretations without statistical backing.
Cross Validation (CV) and Resampling
Holdout Validation

Split data into training set (e.g., 80%) and test set (20%). Simple but high variance — performance depends heavily on the split.

K-Fold Cross Validation

Divide dataset into K equal folds. Train K times, each time using (K-1) folds for training and 1 fold for testing. Average the K results.

  1. Shuffle dataset
  2. Split into K equal folds
  3. For each fold i: train on all folds except i, test on fold i
  4. Average the evaluation metrics across all K folds
Example (K=5): 10 samples split into 5 folds of 2 each. Fold 1 used for test, Folds 2-5 for train. Then Fold 2 for test, etc. Final accuracy = average of 5 fold accuracies.
Stratified K-Fold Cross Validation

Like K-Fold but ensures each fold has the same class distribution as the full dataset. Crucial for imbalanced datasets.

When to use Stratified K-Fold: When class distribution is imbalanced (e.g., 90% class A, 10% class B). Without stratification, some folds might have no minority class examples.
Stratified K-Fold Example
800 samples: 720 Class 0, 80 Class 1. Use 4-fold stratified CV.
Ratio Class 1: 80/800 = 10%
Each fold = 800/4 = 200 samples
Class 1 per fold = 10% × 200 = 20 Class 1 samples per fold
Bootstrapping

Create new datasets by sampling with replacement from the original dataset. Each bootstrap sample is likely to have some repeated and some missing records. Used in ensemble methods like Random Forest.

Measuring Classifier Performance
Confusion Matrix Components
Predicted + Predicted - Actual + TP (True Pos) FN (False Neg) Actual - FP (False Pos) TN (True Neg)
Performance Metrics Formulas
Accuracy = (TP + TN) / (TP + TN + FP + FN) Precision = TP / (TP + FP) ← "Of predicted positives, how many correct?" Recall = TP / (TP + FN) ← "Of actual positives, how many found?" (Sensitivity, True Positive Rate) F1-Score = 2 × (Precision × Recall) / (Precision + Recall) (Harmonic mean — balances precision and recall) Specificity (TNR) = TN / (TN + FP) FPR (False Positive Rate) = FP / (FP + TN)
PRECISION

Use when false positives are costly. Example: spam filter — marking a legitimate email as spam (FP) is bad.

RECALL

Use when false negatives are costly. Example: cancer detection — missing a cancer case (FN) is dangerous.

F1-SCORE

Use when you need to balance precision and recall, especially with imbalanced datasets. Fraud detection.

Paired t-Test

Used to determine if the performance difference between two classifiers on the same dataset is statistically significant.

Steps: 1. Compute differences dᵢ = accuracy_A(i) - accuracy_B(i) for each fold 2. Compute mean: d̄ = (1/n) Σdᵢ 3. Compute std dev: s = √[Σ(dᵢ - d̄)² / (n-1)] 4. Compute t-statistic: t = d̄ / (s / √n) 5. Compare |t| to critical t-value at (n-1) degrees of freedom Decision: |t| ≥ t_critical → Reject H₀ (significant difference exists) |t| < t_critical → Fail to reject H₀ (no significant difference)

Ensemble Learning — Introduction

Ensemble learning combines multiple models (weak learners) to produce a stronger, more accurate and robust model. The core idea: "the wisdom of the crowd is greater than any individual expert."

BIAS

Error from overly simplistic assumptions. High bias → underfitting. Model is too simple to capture the true pattern.

Example: True: y=x², Model: y=ax+b → consistent underprediction

VARIANCE

Error from sensitivity to fluctuations in training data. High variance → overfitting. Model learns noise.

Example: Complex decision tree perfectly fits training data but fails on test data

Ensemble Methods: Bagging → reduces variance | Boosting → reduces bias | Stacking → improves predictions by combining diverse models
Bagging (Bootstrap Aggregating) & Random Forest
Bootstrapping

Create multiple datasets by sampling with replacement from the original dataset. Each bootstrap sample is approximately the same size as the original but with repeated and missing instances.

Original: [1,2,3,4,5] Bootstrap Sample 1: [1,3,4,4,5] ← 4 appears twice, 2 missing Bootstrap Sample 2: [2,2,3,5,1] ← 2 appears twice, 4 missing Bootstrap Sample 3: [5,3,1,3,4] ← 3 appears twice, 2 missing
Bagging Algorithm
  1. Create B bootstrap samples from training data
  2. Train a separate model (usually decision tree) on each bootstrap sample in parallel
  3. For classification: take majority vote across all B models
  4. For regression: take average of all B model predictions

Goal: Reduce variance. Particularly effective for high-variance models (decision trees).

Bagging vs Pasting: Bagging samples with replacement; Pasting samples without replacement.

Random Forest

Random Forest extends bagging with an additional feature randomness step: at each split, only a random subset of features is considered (not all features). This further decorrelates the trees and reduces overfitting.

  • Bootstrapping: Each tree trained on a different random sample
  • Feature Randomness: Random subset of features at each split (usually √n features for classification)
  • Ensemble: Final prediction by majority vote (classification) or average (regression)
Boosting & AdaBoost

Boosting trains models sequentially. Each new model focuses more on the mistakes of the previous one. Combines into a weighted ensemble.

AdaBoost (Adaptive Boosting)
  1. Initialize equal weights for all training samples: wᵢ = 1/N
  2. Train a weak learner (decision stump) on weighted data
  3. Compute error rate: ε = Σwᵢ · 𝕀[yᵢ ≠ ŷᵢ]
  4. Compute model weight: α = 0.5 · ln((1-ε)/ε)
  5. Update sample weights: wᵢ ← wᵢ · exp(-αyᵢĥ(xᵢ)); then normalize
  6. Misclassified samples get higher weight; correctly classified get lower weight
  7. Repeat until convergence or max iterations
  8. Final prediction: H(x) = sign(Σ αₜ·hₜ(x))

Stacking & Voting
Stacking

Train multiple diverse base models, then use their predictions as inputs to a meta-model (also called blender) that learns how to best combine them.

  1. Train base models (e.g., Decision Tree, SVM, Logistic Regression) on training data
  2. Generate out-of-fold predictions from each base model
  3. Create a new "meta-dataset" with these predictions as features
  4. Train the meta-model on this new dataset
  5. For new data: pass through base models, then through meta-model for final prediction
Voting
HARD VOTING

Each classifier votes for a class label. The class with the most votes wins. Example: 3 classifiers predict [Cat, Dog, Cat] → Cat wins (2 vs 1).

SOFT VOTING

Average the predicted probabilities across classifiers. More informative — uses confidence of each classifier. Example: Avg Prob(Yes)=0.67 > Avg Prob(No)=0.33 → predict Yes.

Unsupervised Learning & K-Means Clustering

Clustering groups similar data points together without predefined labels. K-Means is the most popular clustering algorithm.

K-Means Algorithm
  1. Initialization: Choose K data points as initial centroids (randomly or K-Means++)
  2. Assignment: Assign each point to the nearest centroid using Euclidean distance
  3. Update: Recalculate each centroid as the mean of all points assigned to it
  4. Repeat steps 2-3 until centroids don't change (convergence)
Euclidean Distance: d(A,B) = √((x₂-x₁)² + (y₂-y₁)²) Manhattan Distance: d(A,B) = |x₂-x₁| + |y₂-y₁| New Centroid: μₖ = (1/|Cₖ|) Σ xᵢ (mean of all points in cluster k) Objective: Minimize WCSS = Σₖ Σᵢ∈Cₖ ||xᵢ - μₖ||²

Dimensionality Reduction — PCA

Principal Component Analysis (PCA) is a linear technique that reduces dimensionality by finding new axes (principal components) that capture the maximum variance in the data. PCs are orthogonal (uncorrelated).

Curse of Dimensionality

More features ≠ always better. Too many features cause: overfitting, increased computation, sparse data. PCA addresses this.

Steps for PCA
  1. Standardize the data (subtract mean, divide by std dev)
  2. Compute the Covariance Matrix: C = (1/(n-1)) XᵀX (on centered data)
  3. Compute Eigenvalues and Eigenvectors of the covariance matrix
  4. Sort eigenvalues in descending order — larger eigenvalue = more variance captured
  5. Select top k eigenvectors (principal components) based on desired explained variance (e.g., 95%)
  6. Project data onto the new k-dimensional subspace
Covariance Formula: Cov(X,Y) = Σ(xᵢ - x̄)(yᵢ - ȳ) / (n-1) Covariance Matrix for 2 features X₁, X₂: C = [[Var(X₁), Cov(X₁,X₂)], [Cov(X₂,X₁), Var(X₂)]] Eigenvalue equation: C·v = λ·v det(C - λI) = 0 ← Characteristic equation Projection: Z = X · V_k (X = centered data, V_k = top k eigenvectors)

Introduction to Reinforcement Learning

RL is a type of ML where an agent learns to make decisions by interacting with an environment through trial and error. Unlike supervised learning, there's no labeled training data — the agent receives reward signals to guide learning.

Key Components

Learning Cycle
  1. Agent observes current state of environment
  2. Agent selects an action based on its current policy
  3. Action is applied to the environment
  4. Environment responds with a new state and a reward
  5. Agent updates its policy based on this experience
  6. Goal: Learn optimal policy π* that maximizes expected cumulative reward
Model-Based vs. Model-Free RL
MODEL-BASED RL

Agent knows or learns the environment's transition model P(s'|s,a) and reward function R(s,a). Uses this model to plan ahead.

When to use: Static/fixed environments with known physics (chess, simulators)

Examples: Dynamic Programming, Value Iteration, Policy Iteration

MODEL-FREE RL

Agent does NOT know the environment model. Learns directly from experience by interacting with the environment.

When to use: Dynamic, unknown environments (autonomous driving, real robots)

Examples: Q-Learning, SARSA, Actor-Critic

Markov Decision Process (MDP)

MDP is the mathematical framework for RL problems. It formalizes the environment as a tuple (S, A, P, R, γ).

MDP Tuple: (S, A, P, R, γ) S = State space (set of all possible states) A = Action space (set of all possible actions) P = Transition probability: P(s'|s,a) = P(next state = s' | state=s, action=a) R = Reward function: R(s,a,s') γ = Discount factor (0 ≤ γ ≤ 1) — how much future rewards matter Markov Property: P(s'|s,a,history) = P(s'|s,a) (The future depends only on the current state, not the past)
Bellman Equations
State Value Function V(s): V^π(s) = Σₐ π(a|s) Σₛ' P(s'|s,a)[R(s,a,s') + γV^π(s')] Optimal State Value: V*(s) = max_a Σₛ' P(s'|s,a)[R(s,a,s') + γV*(s')] Q-Value Function: Q^π(s,a) = Σₛ' P(s'|s,a)[R(s,a,s') + γ Σₐ' π(a'|s')Q^π(s',a')]

Q-Learning

Q-Learning is a model-free, off-policy RL algorithm that learns the optimal action-value function Q*(s,a). It updates Q-values using the Bellman equation without needing to know the environment model.

Q-Learning Update Rule
Q(s,a) ← Q(s,a) + α[R + γ·max_a' Q(s',a') - Q(s,a)] Where: Q(s,a) = current Q-value of (state, action) α = learning rate (0 < α ≤ 1) — how fast to update R = immediate reward received γ = discount factor — importance of future rewards max_a' Q(s',a') = maximum Q-value achievable from next state s' s' = next state after taking action a TD Target = R + γ·max_a' Q(s',a') TD Error = TD Target - Q(s,a)

ε-Greedy Policy (Exploration vs. Exploitation)
ε-Greedy Policy: With probability ε → choose random action (EXPLORATION) With probability 1-ε → choose best known action (EXPLOITATION) Common strategy: Start with high ε (lots of exploration) Gradually decrease ε over time (more exploitation)
Exploration vs. Exploitation Tradeoff: Exploration discovers new potentially better actions. Exploitation uses current best-known action. Too much exploitation → get stuck in local optimum. Too much exploration → never converge. ε-greedy balances this.
Temporal Difference (TD) Learning

TD learning blends ideas from Monte Carlo (learns from actual experience) and Dynamic Programming (updates from estimates). It updates after each step, not after a complete episode.

Active Reinforcement Learning

In Active RL, the agent actively selects actions to explore the environment and improve its policy. It is not just passively learning from given data — it makes decisions to gather information.

  • Intentional Exploration: Agent deliberately tries new actions
  • Exploration Methods: ε-greedy, Boltzmann exploration, Upper Confidence Bound (UCB)
  • Learns online and continuously from experience
  • Suitable for unknown, dynamic environments (robot navigation, game AI)

Generalization in Reinforcement Learning

Basic tabular RL treats every state individually — infeasible for large/continuous state spaces. Generalization allows the agent to transfer knowledge from seen states to unseen similar states.

  • Uses function approximators instead of lookup tables: Linear models, Neural Networks (Deep RL), Decision Trees
  • Benefits: Faster learning, better scalability, handles continuous state spaces
  • Example: A robot trained in one maze learns "avoid walls" → applies this in a new maze it has never seen before
  • Deep RL (e.g., DQN): Uses a neural network to approximate Q(s,a) for all possible states
Generalization vs. Memorization: Tabular Q-learning memorizes Q-values for each (s,a) pair — works for small discrete spaces. Function approximation generalizes — essential for games like Chess, Go, or real-world robotics.

Q1a (5 marks): Analyze when machine learning is the appropriate solution to a problem. Give examples.
Machine learning is appropriate when: (1) Existing solutions require many fine-tuned rules — ML simplifies code and adapts automatically (e.g., spam filters that update without manual rule changes). Q1b (5 marks): Explain the concepts of batch learning and online learning.
Batch Learning: The model is trained using the entire dataset at once. Once trained, it is deployed and does not learn anymore (offline learning). If new data arrives, a full retraining cycle is needed. Resource-intensive. Used when data is static (e.g., training a weather prediction model annually). Online Learning: The model updates incrementally as each new data instance (or mini-batch) arrives. Ideal for streaming data (stock prices, sensor data). Key parameter: learning rate — controls adaptation speed. High rate → fast adaptation but forgets old data. Low rate → stable but slow. Advantages: handles huge datasets (processes and discards), adapts to changes, low memory. Example: stock price predictor that updates after each trade.

Q6a (10 marks): Stratified k-fold CV vs standard k-fold CV. Imbalanced data preference. Class 1 samples per fold with 800 samples (720 Class 0, 80 Class 1), 4-fold.
Standard K-Fold CV: Randomly splits dataset into K equal folds without considering class distribution. Problem: with imbalanced data, some folds might have very few (or no) minority class examples — biased model.

Stratified K-Fold CV: Ensures each fold has approximately the same class proportion as the original dataset. For each class, samples are divided into K strata and combined to form folds.

Differences: Stratified preserves class ratios; standard does not. Stratified preferred for imbalanced datasets; standard works for balanced ones.

Imbalanced (90% A, 10% B): Use Stratified K-Fold. Without stratification, some folds might have 0% Class B → model never sees minority class in some training/test cycles → unreliable evaluation.

Q7b (10 marks): Dataset with 200 features, model overfitting. How can PCA help? Describe PCA process.
How PCA helps: With 200 features, the model may overfit due to the curse of dimensionality — too many features relative to training samples. PCA reduces 200 features to k principal components (e.g., 20-30) while retaining 95%+ of variance. This reduces: model complexity (fewer parameters), overfitting (regularization effect), training time, memory usage, and noise (by discarding low-variance components).

PCA Process:
1. Standardize data: Subtract mean, divide by std dev for each feature.
2. Compute Covariance Matrix: C = (1/n-1)XᵀX (200×200 matrix).
3. Compute Eigenvalues & Eigenvectors of C.
4. Sort eigenvalues in descending order — plot scree plot.
5. Select k components: Choose k where cumulative explained variance ≥ 95%. Example: if first 20 components explain 95% variance, use k=20.
6. Transform data: Project 200D data onto 20D subspace.
7. Train model on reduced 20D dataset — much less overfitting risk.
Q8a (10 marks): Explain Random Forest algorithm, how it reduces overfitting vs single decision tree.
Random Forest: An ensemble of decision trees using bagging + feature randomness. Each tree trained on a different bootstrap sample, and at each node, only a random subset of features is considered for splitting.

Steps: (1) For each of N trees: draw bootstrap sample from data. (2) Build decision tree: at each split, randomly select √m features (m=total features); choose best split from these. (3) Prediction: majority vote (classification) or average (regression).

Why it reduces overfitting vs single tree:
- Single decision tree: can grow deep and memorize training data (high variance). Very sensitive to specific training samples.
- Random Forest: Multiple trees are trained on different data subsets → averaging reduces variance. Feature randomness ensures trees are decorrelated (different trees make different errors) → ensemble error is lower than any individual tree. Bootstrap sampling ensures no single outlier dominates all trees.

Advantages: Handles high dimensionality, robust to outliers, provides feature importance. Limitations: Less interpretable than single tree, slower prediction, high memory.
Q8b (10 marks): What is Ensemble Learning? Describe Bagging and Boosting. Compare bias/variance impact.
Ensemble Learning: Technique combining multiple models (weak learners) to produce a stronger model. Core idea: diverse models make different errors; combining them cancels out individual errors. Two main benefits: improved performance and increased robustness.

Bagging (Bootstrap Aggregating): Train multiple models in parallel on different bootstrap samples. Combine by majority vote (classification) or averaging (regression). Effect: Reduces variance (not bias). Best for high-variance models like decision trees. Example: Random Forest.

Boosting: Train models sequentially; each new model focuses on errors of previous. Assign higher weights to misclassified samples. Combine via weighted sum. Effect: Reduces bias (and some variance). Best for weak learners like stumps. Examples: AdaBoost, XGBoost, Gradient Boosting.

Scenarios: Bagging → when model overfits (high variance). Boosting → when model underfits (high bias). Boosting can overfit noisy data; bagging is more robust to noise.

Q10b (5 marks): Define active reinforcement learning. How does it differ from passive?
Active RL: The agent actively selects its own actions to explore the environment and learn the optimal policy. It must balance exploration (trying new actions to gather information) and exploitation (using current knowledge). Implemented via ε-greedy, UCB, or Thompson sampling. Examples: Q-Learning, SARSA.

Passive RL: The agent follows a fixed, externally given policy and just evaluates how good that policy is. No control over which actions are taken. Example: Learning from demonstrations (imitation learning).

Q10c (10 marks): Explain exploration vs exploitation tradeoff. How does ε-greedy manage it?
Exploration: Trying new, unknown actions to discover potentially better rewards. Risk: may waste time on bad actions. Essential early in learning when the agent knows little.

Exploitation: Choosing the action currently known to give the highest reward. Risk: may get stuck in a local optimum, miss better strategies.

The Tradeoff: Too much exploration → never converges to optimal policy. Too much exploitation → misses better strategies discovered through exploration. The agent must balance both throughout learning.

ε-Greedy Policy:
With probability ε → choose a random action (explore)
With probability (1-ε) → choose argmax Q(s,a) (exploit)

Common strategy: Start with ε=1.0 (pure exploration) and decay it over time: ε = ε₀ · e^(-decay_rate × episode). As the agent learns more, it explores less and exploits more. The decay rate controls how quickly the agent transitions from explorer to exploiter.

Example: In early episodes of Q-learning on a maze, ε=0.9 → mostly random moves. After 1000 episodes, ε=0.1 → agent mostly follows learned optimal path but occasionally tries new routes.

Q4b (10 marks): Explain ID3 algorithm. How are entropy and information gain used?
Already covered extensively in Unit 2 Decision Trees section. Key points: ID3 (Iterative Dichotomiser 3) builds tree top-down by greedily selecting the attribute with maximum Information Gain at each step. Entropy measures impurity of a node. Information Gain measures how much an attribute reduces entropy. Process: Calculate H(S) → for each attribute, calculate weighted entropy after split → IG = H(S) - weighted entropy → select attribute with max IG as node. Recursively repeat for sub-trees. Stop when node is pure (H=0) or no attributes left.
